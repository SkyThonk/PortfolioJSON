3. **Commit Messages**
   - Use clear, descriptive messages
   - Format: `type(scope): description`
   - Examples:
     - `feat(projects): add filter by technology`
     - `fix(navbar): resolve mobile menu overflow`
     - `docs(readme): update installation steps`
     - `style(hero): improve responsive layout`

4. **Testing**
   ```bash
   # Run tests
   npm test
   
   # Run linting
   npm run lint
   ```

5. **Before Submitting**
   - [ ] Code builds without errors
   - [ ] All tests pass
   - [ ] No console errors
   - [ ] Responsive on mobile/tablet/desktop
   - [ ] Updated documentation if needed
   - [ ] Added comments for complex logic

#### Pull Request Process

1. **Update Documentation**
   - Update README.md if adding features
   - Update PORTFOLIO-DATA-GUIDE.md if changing JSON structure

2. **Create Pull Request**
   - Clear title describing the change
   - Description with:
     - What changed
     - Why it changed
     - Screenshots/videos if UI changed
     - Link to related issue

3. **Code Review**
   - Respond to feedback
   - Make requested changes
   - Keep discussions professional

4. **After Merge**
   - Delete your branch
   - Pull latest main branch
